# Attribution — vendored skills

These skills are mirrored verbatim from [`nexu-io/open-design`](https://github.com/nexu-io/open-design)
`skills/` for provenance and reference. Per that repo, **skills are Apache-2.0 unless a skill's own
`LICENSE` says otherwise** (the notable exception is `guizang-ppt/`, MIT, bundled from
[op7418/guizang-ppt-skill](https://github.com/op7418/guizang-ppt-skill)).

Each skill records its original source in its `SKILL.md` frontmatter under `od.upstream`. open-design
itself curated many of these from upstream authors (Anthropic, Google Labs, ComposioHQ, individual
authors, etc.); see each skill's `upstream` and the generated [`CATALOG.md`](./CATALOG.md).

medesign does not modify the vendored copies. The **curated, code-first adaptations** medesign actually
runs live under the repo's top-level `skills/<name>/` and are medesign-authored (they reference these
vendored skills as design sources). See [`docs/harness-engine.md`](../../../docs/harness-engine.md) and
[`docs/skills.md`](../../../docs/skills.md).

This vendoring is consistent with the Apache-2.0 license (attribution preserved; `NOTICE` at the repo
root notes the broader open-design adaptation).
